| **Euphoria → C言語トランスレータは任意のEuphoriaプログラムと等価なC言語ソースコードへ翻訳・変換します。**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| トランスレータには**Windows**, **DOS**, **Linux** および **FreeBSD**版があります。 EuphoriaプログラムをC言語ソースコードに翻訳後、対応するコンパイラのうち1を使用してコンパイルとリンクができます。これにより普通にEuphoriインタプリタを使用するのと比べて非常に高速に動作する実行可能ファイルが生成されます。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| トランスレータ\*自身\*は動作環境ごとに翻訳/コンパイルして実行可能ファイルにすることができます。トランスレータはインタプリタのフロントエンドの一部であり翻訳/コンパイルで使用されます。トランスレータのソースコードはeuphoria¥sourceにあります。これは全てEuphoriaで記述されています。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| []{#compsupp}\                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| 2. 対応するC言語コンパイラ                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| 現在のところ**トランスレータ**は**Linux**または**FreBSD**上の[GNU C](http://gcc.gnu.org/)や、**DOS**上の [OpenWatcom C](http://www.openwatcom.org)または[DJGPP C](http://www.delorie.com/djgpp/)の双方、**Windows**上のOpenWatcom C, [Lcc](http://www.q-software-solutions.de/)または[Borland C++ Compiler 5.5](http://www.codegear.com/downloads/free/cppbuilder)の3種類で動作します。これらは全て無償提供されています。GNU CはLinuxまたはFreeBSDシステムにおいて標準でインストールされています。その他は各ウェブサイトからダウンロードできます。WindowsではLccよりOpenWatcom CまたはBorland C++ Compiler 5.5の使用を強く推奨します。Lccの開発は活発であり改良され続けてはいるものの、いくつかバグがあるため巨大なWindowsプログラムを正常にコンパイルすることが困難です。OpenWatcom CまたはBorland C++ Compiler            |
| 5.5では正常にコンパイルできます(**訳注:この記述は当時(2005年以前)のものであり現在のバージョン(和訳時点では2009.12.02)では既に改善されているものと思われます**)。OpenWatcom Cのほうがやや小さくて高速な実行ファイルを生成できますが、大量のC言語ソースをコンパイルすときはBorland C++ Compiler 5.5のほうが高速です。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| OpenWatcom CのDOS32版には[CauseWay](http://www.devoresoftware.com) DOSエクステンダとファイル圧縮ツールが含まれています。現在ではCauseWayは無償のオープンソースです。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **emake.bat** および **objfiles.lnk** は自動でCauseWay DOSエクステンダをリンクします。その他DOS4GWのようなDOSエクステンダでは**トランスレータ**は正常に動作しません。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **トランスレータ**は環境変数 \"WATCOM\", \"LCC\", \"BORLAND\" または \"DJGPP\" とディレクトリの**パス**の両方を調べます。 これは、コンパイラとリンカを適切に起動させるためのバッチファイル**emake.bat** を生成します。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **注意事項:**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| - Watcomとは違いDJGPPではDOS下位メモリーを他の同一セグメント内に配置しません。Watcom版**Euphoriaインタプリタ**または**トランスレータ**向けに書かれた機械語ルーチンはDJGPPでは動作せず、ビデオメモリーといった下位メモリーにアクセスすると恐らくクラッシュします。Euphoriaの**peek()**, **poke()**, **mem_copy()**, **mem_set()**などはDJGPPの特殊な下位メモリーアクセスマクロを**トランスレータ**で使用することにより正常に動作するようになります。機械語ルーチンをDJGPPへ移植することができますが、下位メモリーのアクセスを可能にするにはDJGPPの説明書を調べる必要があります。                                                                                                                                                                                                                                             |
| - DJGPPでは長いファイル名の読み書き、作成に完全対応していますが、Watcom Cでは作成は対応していません。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| - DJGPP版トランスレータで生成したプログラムでマウスは使用できません。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| - 主にAllegroグラフィックスライブラリはDJGPPで使用され、多くの場合においてWatcom標準グラフィックスライブラリより高速です。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| - DJGPPでは数種類以上のテキストモードに対応してます。すなわち35行モードがあります。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| - DJGPPでは利用者がCtrl+Cを入力することにより何時でもプログラムを強制終了することができます。.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| - Lccでは**lock_file()** および **unlock_file()**の実装を無視します。よってこれらは使用できません。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| - **emake.bat**において**トランスレータ**はLccのオプションに-Oフラグを使用します。このフラグは時としてLccの不安定さの原因となることがあります。もしプログラムが動作しないとき、試しにemake.bat内のコンパイルコマンドからこのフラグを全て削除してください。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| - **emake.bat**によるコンパイルでは警告の出力は無効になります。もし有効にすると、未使用の定義済み変数、未使用の定義済みラベル、未定義の関数プロトタイプなどで無害なメッセージが表示されることがあります。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| - Windowsでは、Watcomリンカはgraph.libを開くことができないという警告を出すことがありますがgraph.libは使用されないため、これは無視できます。この警告を抑制して非表示にするのは容易ではありません。この問題は最新のOpenWatcomでは既に修正されています。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| - BorlandおよびLccのWindowsコンソール(テキストモード)プログラムではファンクションキーまたは矢印キーを読み取ることができず、Enterキーを2回押す必要があることがあります。この問題はWindows GUIプログラムでは発生しません。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| - 現在のところMicrosoft C++コンパイラ(Windows)には対応していません。しかし **ecw.exe**により生成されたC言語ソースファイルをインポートして、さらにLccまたはBorlandのライブラリファイルをプロジェクトに追加してから細かい問題点を修正するとコンパイル/リンクができます。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

|            BorlandおよびLccでは標準環境変数がないため、**トランスレータ**は**PATH**変数からコンパイラのディレクトリと思われるものを検索します。例えば標準的な場所を検索します: ..¥LCC, ..¥BCC.., ..¥Borland.. など。もし標準以外の場所にインストールしたときはインストールディレクトリの名前を変更しなければならないときがあります。Windows XPでPATH変数を編集するには、次の手順でアイコンをクリックします: スタートメニュー / コントロールパネル/ パフォーマンスとメンテナンス / システム / 詳細設定 / 環境変数 / ユーザ環境変数 \... そしてPATHを選んで値を編集します。次を末尾に挿入します(挿入前のPATH文字列の末尾に;がないときは末尾に;を付加してください): C:¥BORLAND¥BCC55¥BIN; または C:¥LCC¥BIN; 最後にOKを数回クリックしてウィンドウを閉じます。次回、DOSウィンドウを開いたときから新しい設定が有効になります。   |
